Applied Rules in Synthetic Studio

How conditional rules work in the schema editor and how to add one.

What rules are

Each row is normally generated independently per column — status is random, cancelled_at is random, etc. A rule is a conditional override that fires after the base generator runs:

IF one or more conditions match → THEN override one column on that row.

That's how you make rows look internally consistent. Without rules, a row can end up with status="active" but cancelled_at="2024-03-15" — clearly fake. With a rule, you can say "if status != 'cancelled', then cancelled_at is null."

Available actions (the THEN side)

Action What it does Example value
Set to NULLForce the column to null(no value)
Ensure not nullForce the column to non-null(no value)
Set constant valueOverride with a fixed value"cancelled" or 42
Pick from listPick uniformly from a listactive, inactive, pending
Set number rangeOverride range for numbers0-100
Derive offset from columnDate/number math relative to another columncreated_at, 1, 30, days
Compute expressionSimple arithmetic between two columnsquantity * unit_price

How to add a rule

  1. In the schema editor (Tables tab), look at the right-side Conditions Applied sidebar.
  2. Click + Add Rule at the bottom.
  3. STEP 1 Conditions (IF): pick a column, an operator (==, !=, <, contains, is null, …), and a value. Click "+ Add Condition" to AND another condition.
  4. STEP 2 Action (THEN): pick the action and the target column. Fill the value field if the action needs one — placeholder shows the format.
  5. Click Save Rule.
